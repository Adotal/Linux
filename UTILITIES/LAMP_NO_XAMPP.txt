Debes habilitar algunas extensiones en tu archivo php.ini para que phpMyAdmin funcione correctamente:
Abre el archivo: sudo nano /etc/php/php.ini
Busca y descomenta (quita el ; al inicio) las siguientes líneas:
extension=bz2
extension=iconv
extension=mysqli
extension=pdo_mysql
extension=gettext
3. Configurar Apache para reconocer phpMyAdmin
Tienes que decirle a Apache dónde encontrar los archivos de phpMyAdmin. La forma más limpia es crear un archivo de configuración separado:
Crea el archivo de configuración:
bash
sudo nano /etc/httpd/conf/extra/phpmyadmin.conf
Usa el código con precaución.

Pega el siguiente contenido dentro:
apache
Alias /phpmyadmin "/usr/share/webapps/phpMyAdmin"
<Directory "/usr/share/webapps/phpMyAdmin">
    DirectoryIndex index.php
    AllowOverride All
    Options FollowSymlinks
    Require all granted
</Directory>
Usa el código con precaución.

Ahora, dile a Apache que cargue ese archivo. Abre /etc/httpd/conf/httpd.conf:
bash
sudo nano /etc/httpd/conf/httpd.conf
Usa el código con precaución.

Ve al final del archivo y añade esta línea:
apache
Include conf/extra/phpmyadmin.conf
Usa el código con precaución.

4. Reiniciar Apache
Para que todos los cambios surtan efecto, reinicia el servicio:
bash
sudo systemctl restart httpd

Tip de seguridad (Solución al error de "Access Denied")
En Arch Linux, por defecto MariaDB usa el plugin unix_socket para el root, lo que significa que solo puedes entrar desde la terminal con sudo. Para entrar a phpMyAdmin como root, debes cambiar el método de autenticación:
Entra a MariaDB desde la terminal: sudo mariadb
Ejecuta estos comandos:
sql
ALTER USER 'root'@'localhost' IDENTIFIED VIA mysql_native_password USING PASSWORD('12345');
FLUSH PRIVILEGES;
EXIT;
